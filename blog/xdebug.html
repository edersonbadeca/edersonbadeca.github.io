Xdebug melhor ferramenta de debug para PHP

print_r();die nunca mais

Junho 26, 2015

A maioria dos programadores PHP que estão começando ou até mesmo os mais experientes, as vezes sofrem para encontrar um erro na lógica, bug ou rastrear alguma transação. Isto deve-se ao fato de utilizar comandos de print como var_dump(), print_r(), echo ou qualquer comando equivalente para validar o valor das variáveis.

O xdebug, é uma ferramenta que permite incluir breakpoints no seu código e executar a depuração. Existem outras aplicações para esta ferramenta também, um exemplo clássico é de profilling, ou seja, você quer saber o desempenho da sua aplicação, tempo de processamento de cada transação dentro do sistema e etc..

Bora começar?

Bom, eu estou usando nesse momento o meu laptop com ubuntu, por isto vou colocar os comandos para instalar no ubuntu okay? Se vocês precisarem de outro sistema operacional, me avise no twitter blza?

Vou presumir que você já tenha instalado seu ambiente de desenvolvimento PHP. Eu particularmente não gosto de ferramentas como wamp,lamp,xampp ... gosto de ter controle do que eu estou usando.

Caso você esteja usando alguma dessas ferramentas, basta habilitar o xdebug nas opções do PHP. Se você estiver instalado o php5 manualmente igual a mim então o próximo passo é instalar a biblioteca xdebug no seu env, abra o terminal e digite:

$ apt-get install php5-xdebug
Após este commando, você já tera instalado o xdebug na maquina :D
Se quiser conferir, ainda no terminal, execute o commando:

php -i |grep xdebug

Este comando irá retorna as configurações definidas por padrão ao xdebug.

Um dos resultados deste comando deve trazer o caminho de arquivo chamado : 20-xdebug.ini
Abra este arquivo com (permissões de sudo) e acrescente as seguintes configurações: 

zend_extension=/usr/lib/php5/20121212/xdebug.so
xdebug.remote_enable=1
xdebug.remote_handler=dbgp
xdebug.remote_mode=req
xdebug.remote_host=127.0.0.1
xdebug.remote_port=9000

Atenção

O caminho da variável zend_extension pode mudar, dependendo da versão que você esta usando. Geralmente fica no diretório:
/usr/lib/php5/[um-numero]/xdebug.so

Pronto! você já pode incluir breakpoints nos seus códigos com a sua IDE ou editor de texto favorito. Eu quando estou desenvonvendo, costumo utilizar o vim, mas existem boas IDEs para PHP, como por exemplo:
NetBeans e Phpstorm.
